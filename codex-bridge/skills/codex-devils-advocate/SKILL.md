# Codex Devil's Advocate スキル

## 概要

Claudeの提案・設計・判断に対して、Codexに意図的に反論させるスキル。
他のセカンドオピニオン系スキルが「独立分析で見落としを補う」のに対し、
このスキルは「提案の弱点を意図的に攻撃する」ことで提案を強化する。

## 他スキルとの違い

| | セカンドオピニオン系 | Devil's Advocate |
|---|---|---|
| Claudeの分析をCodexに渡すか | 渡さない | **渡す（意図的に）** |
| Codexの役割 | 独立した分析者 | 反論者・批判者 |
| 目的 | 見落としの発見 | 提案の弱点を攻撃して強化 |

## 実行手順

### 1. Claudeの提案をまとめてユーザーに提示

まずClaudeの提案・設計・判断を明確にまとめ、**シフィさんに先に提示する**。
（他スキルでは内部保持するが、このスキルではユーザーに見せる）

### 2. Codexセッションの開始

`mcp__codex__codex` を呼び出す。

- **developer-instructions**:

```
あなたはDevil's Advocate（悪魔の代弁者）です。
別のAIアシスタント（Claude）が提案を作成しました。
あなたの役割はこの提案に対して意図的に反論・批判を行うことです。

- 提案の弱点、リスク、見落としを徹底的に指摘する
- 「うまくいかないケース」を具体的に挙げる
- 暗黙の前提を洗い出し、それが崩れた場合の影響を示す
- 代替案があれば提示する（ただし反論が主目的）
- 建設的であること（攻撃だけでなく改善の方向性も示す）
- 日本語で回答する

重要: あなたの役割は「反対するために反対する」ことではなく、
提案の品質を高めるために弱点を露出させることです。
```

- **prompt**:

```
以下のClaudeの提案に対してDevil's Advocateとして反論してください。

【Claudeの提案】
{claude_proposal}

【提案の背景・コンテキスト】
{context}

この提案の弱点、リスク、見落としを指摘してください。
各指摘に対して:
1. 具体的な問題シナリオ
2. 影響の深刻度
3. 改善の方向性
を含めてください。
```

- **cwd**: 現在のプロジェクトディレクトリ
- **sandbox**: `read-only`

### 3. 反論への対応

Codexの反論を受けて、Claudeが以下を整理する:

```
═══ Devil's Advocate 結果 ═══

【Claudeの提案】
  {proposal_summary}

【Codexの反論】
  1. {criticism_1}
     深刻度: {severity}
     → 改善の方向性: ...

  2. {criticism_2}
     深刻度: {severity}
     → 改善の方向性: ...

【Claudeの回答】
  反論1に対して: {defense_or_acceptance}
  反論2に対して: {defense_or_acceptance}

【提案の修正案】（修正が必要な場合）
  - 変更点1: ...
  - 変更点2: ...

【シフィさんへの確認事項】（判断が必要な場合）
  → ...
```

### 4. 提案の強化

- 正当な反論に対しては提案を修正する
- 反論が的外れな場合はその理由を説明する
- 判断が分かれる場合はシフィさんに委ねる

## 活用タイミング

- 重要な設計判断の前
- リスクの高い技術選定時
- 「本当にこれで良いのか」を確認したい時
- PRのセルフレビュー補強として

## 注意事項

- このスキルではClaudeの提案をCodexに**意図的に渡す**（独立性よりも批判的検証を優先）
- Codexの反論に対してClaudeは防御的にならず、正当な指摘は素直に受け入れる
- 反論の深刻度をClaudeが独断で判断せず、シフィさんにも見解を共有する
